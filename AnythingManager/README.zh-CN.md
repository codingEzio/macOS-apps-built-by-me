# AnythingManager

[English README](README.md)

一个 macOS 菜单栏应用，用来一键启动、停止和监控你的本地开发服务器（`bun run dev`、`next dev`、`python -m http.server` 等），不需要一直开着终端窗口。

## 它是干嘛的（说人话版）

**它解决什么问题：**

你同时在做一个 Next.js 网站和一个 Node API。每天早上你打开两个终端窗口，`cd` 进各自的项目，然后运行 `bun run dev`。这两个终端一直占着屏幕。你不小心关掉一个。你忘了哪个端口对应哪个项目。你又启动了一份，报错 "EADDRINUSE"。然后你用 `lsof` 和 `kill` 去 hunt 那个旧进程。烦死了。

**这个应用做什么：**

1. 你一次性配置好项目（名字、文件夹路径、命令、端口）。
2. 应用住在菜单栏。点一下，按 **Start**，关掉面板。
3. 你的服务器在后台跑。不需要终端窗口。
4. 图标在服务器运行时变 **绿色**，停掉时变 **灰色**。
5. 如果端口被占了 —— 哪怕是你昨天在终端里忘关的旧会话 —— 应用会检测到，杀掉旧进程，启动你的。

## 功能一览

| 功能 | 说明 |
|------|------|
| **一键 Start / Stop / Restart** | 不需要终端。 |
| **智能端口接管** | 端口被占了？应用会杀掉旧监听器，然后启动你的。 |
| **外部进程检测** | 昨天在终端里启动过服务器？应用能看到它，显示 "External"，然后让你点 **Take Over** 接管。 |
| **重建后仍然能控制** | 用 `./restart-app.sh` 重新编译并重启应用。你的服务器继续跑。新应用实例会自动检测到它们。 |
| **按项目查看日志** | 看最近 1 万字的输出。随时清空。 |
| **配置热重载** | 用任何编辑器改 `config.json`，1 秒内应用里就会更新。 |
| **开机自启** | 可选。登录时自动启动。 |
| **点外面自动关闭** | 点桌面或其他应用 —— 面板自动关闭，跟原生 macOS 菜单栏应用一样。 |

## 怎么用

### 第一次使用

```bash
cd AnythingManager
./build-app.sh
open ../Applications/AnythingManager.app
```

### 开发时（重新编译 + 重启）

```bash
cd AnythingManager
./restart-app.sh
```

你的开发服务器继续运行，因为它们是独立的操作系统进程。
新应用会检测到它们，显示 **"Take Over"** 让你重新接管。

### 配置你的项目

打开应用 → 点 **Settings** → 添加项目：

| 字段 | 示例 |
|------|------|
| Name | `my-website` |
| Path | `~/projects/my-website` |
| Command | `bun run dev` |
| Port | `3000` |

也可以直接编辑 `config.json`。目录里提供了 `config.sample.json` 作为模板。

## 项目结构

| 文件 | 作用 |
|------|------|
| `Sources/AppDelegate.swift` | 菜单栏图标、面板设置、点外面关闭 |
| `Sources/ContentView.swift` | 主界面 —— 项目卡片、端口警告条、日志查看器 |
| `Sources/SettingsView.swift` | 设置 —— 编辑项目、开机自启开关 |
| `Sources/ProcessManager.swift` | 启停进程、端口健康检查、配置热重载 |
| `Sources/PortChecker.swift` | 基于 `lsof` 的端口检测，带安全白名单 |
| `Sources/Project.swift` | 可编码的项目模型 |

## 安全说明

- **只杀开发服务器进程。** `killPort()` 会检查进程名是否在白名单里（`node`、`next`、`bun`、`npm`、`python`、`vite` 等）。不匹配的不杀，Chrome、系统服务、未知进程一律跳过。
- **父进程也只杀白名单里的。** 你的 Shell 和 Terminal 永远不会被碰。
- **没有 Shell 注入。** 工作目录通过 `Process.currentDirectoryURL` 设置，而不是把路径嵌进 Shell 命令字符串里。
- **白名单匹配很严格。** 不是简单的"包含"子串，而是要求进程名的最后一段完全匹配，或者是匹配前缀加连字符（例如 `node-sass` 匹配 `node`，但 `anodeb` 不会匹配）。

## 环境要求

- macOS 13+
- Swift 5.9+（Xcode 或命令行工具）

## 常见问题

**"应用没有出现在菜单栏"**
→ 确保你打开的是 `.app` 包，而不是 `.build/debug/` 里的裸二进制文件。

**"点 Start 报错"**
→ 检查 **Path** 是否存在，**Command** 手动运行时是否正常：
  `zsh -l -c "cd <path> && <command>"`

**"显示 Running 但 localhost 没反应"**
→ 确保 Settings 里的 **Port** 跟你的开发服务器实际绑定的端口一致。
